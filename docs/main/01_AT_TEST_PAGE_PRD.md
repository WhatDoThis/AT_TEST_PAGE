## 1. 개요

### 1.1 제품 한 줄 정의

**AT_TEST_PAGE**는 웹과 Android에서 동일한 코드베이스로 동작하는 단일 화면 애플리케이션으로, 설정 파일(`env/config.json`)만으로 이미지 구성·라벨·서비스 메타를 바꿀 수 있는 **이미지 중심 테스트/데모 페이지**다.

### 1.2 배경·목적

- 웹(로컬 개발·리눅스 프로덕션 배포)과 Android(APK/AAB)를 **한 저장소**로 유지한다.
- “Next.js 스타일 라우팅이 있지 않았나?”에 대응하는 **파일 기반 라우팅**을 **Expo Router**로 통일한다.
- 이미지 추가·교체 시 **앱 코드 수정 없이** `config.json`과 `assets/images/`만으로 운영 가능하게 한다.

### 1.3 범위 (In Scope)

| 구분 | 포함 |
|------|------|
| 플랫폼 | 웹(Expo for Web), Android |
| 화면 수 | 단일 메인 페이지 (`app/index.tsx` 수준) |
| 라우팅 | Expo Router, 공통 레이아웃(`_layout.tsx`) |
| 설정 | `env/config.json` 로드 및 앱 제목·포트·이미지 목록 반영 |

### 1.4 비범위 (Out of Scope) — v1.0

- iOS 전용 빌드·스토어 배포(필요 시 후속 버전에서 정의).
- 사용자 인증·서버 저장 API·실시간 협업.
- 이미지 업로드를 앱 내에서 처리하는 CMS 수준 기능(파일은 배포 시 `assets` 또는 정적 경로로 관리).

## 2. 이해관계자·가정

### 2.1 이해관계자

- **개발자**: Expo/EAS 또는 로컬 빌드로 웹·Android 산출물 생성.
- **운영/테스트 담당**: `config.json` 수정으로 화면 문구·이미지 구성 변경.

### 2.2 가정

- Node.js 및 Expo CLI 사용 환경이 준비되어 있다.
- 프로덕션 웹은 리눅스 등에서 정적/Node 호스팅 정책에 맞게 배포한다(배포 파이프라인은 별도 운영 문서).

## 3. 기술 스택 (확정)

| 항목 | 선택 | 근거 |
|------|------|------|
| 프레임워크 | **Expo SDK (React Native)** | 웹 + Android 단일 코드베이스, TypeScript |
| 라우팅 | **Expo Router** | Next.js와 유사한 파일 기반 라우팅, 웹/앱 동일 패턴 |
| 웹 | **Expo for Web** (`npx expo start --web` 등) | 로컬 개발·프로덕션 웹 배포 |
| Android | **EAS Build** 또는 로컬 빌드 | APK/AAB |
| 언어 | **TypeScript** | 타입 안정성·도구 호환 |
| 설정 | **`env/config.json`** | 포트, base URL, 이미지 디렉터리, 앱 타이틀, 이미지 메타 |

> 정리: “웹+앱에서 Next 느낌의 라우팅” → **Expo Router**가 해당 역할을 수행한다.

## 4. 아키텍처·폴더 구조

### 4.1 디렉터리 (목표 구조)

```
AT_TEST_PAGE/
├── app/                         # Expo Router
│   ├── _layout.tsx              # 공통 레이아웃 (상단 헤더: 앱 타이틀)
│   └── index.tsx                # 메인 단일 페이지
├── components/
│   ├── ImageCarousel.tsx        # 메인 이미지 슬라이드(우측 전환 애니메이션)
│   ├── ImageGallery.tsx         # 이미지 전체보기 카드 섹션
│   └── ToggleButton.tsx         # 섹션/영역 보이기·감추기
├── assets/
│   └── images/                  # 기본 샘플 이미지
│       ├── default.png
│       ├── img_02.png
│       └── img_03.png
├── env/
│   └── config.json              # 런타임·빌드 전 참조 설정
├── app.json                     # Expo 앱 설정
├── tsconfig.json
└── package.json
```

### 4.2 구조 원칙

**Front** (`app/` + `components/`) + **Image** (`assets/images/`) + **Env** (`env/`) — 세 축으로 단순화한다.

## 5. 설정 명세 (`env/config.json`)

### 5.1 스키마 (v1.0)

```json
{
  "port": 3000,
  "base_url": "http://localhost:3000",
  "image_dir": "./assets/images",
  "app_title": "AT_TEST_PAGE",
  "images": [
    { "id": 1, "filename": "default.png", "label": "메인 이미지" },
    { "id": 2, "filename": "img_02.png", "label": "이미지 2" },
    { "id": 3, "filename": "img_03.png", "label": "이미지 3" }
  ]
}
```

### 5.2 필드 정의

| 필드 | 타입 | 설명 |
|------|------|------|
| `port` | number | 로컬 개발 서버 등에서 참고할 포트(도구·문서와 정합; 실제 바인딩은 실행 스크립트와 일치 필요) |
| `base_url` | string | 웹 기준 URL(링크·메타·디버그용; 배포 시 교체) |
| `image_dir` | string | 이미지 파일 기준 상대 경로 |
| `app_title` | string | 헤더·문서 타이틀 등에 표시할 앱 이름 |
| `images` | array | 갤러리·캐러셀에서 사용할 이미지 메타 목록 |
| `images[].id` | number | 고유 식별자(표시 순서 정렬에 사용 가능) |
| `images[].filename` | string | `image_dir` 기준 파일명 |
| `images[].label` | string | UI에 표시할 한글/영문 라벨 |

### 5.3 설정만으로 바꿀 수 있어야 하는 것

- 앱 상단 제목 문자열.
- 캐러셀·갤러리에 노출되는 이미지 목록 및 라벨.
- (배포 환경에 맞춘) `base_url` / `port` 문서화 값.

코드 변경 없이 위만 수정해도 동작해야 한다는 것이 **수용 기준**에 포함된다.

## 6. 사용자 시나리오

1. 사용자가 앱/웹을 실행하면 **고정 헤더**에 `app_title`이 표시된다.
2. **메인 캐러셀**에 현재 선택된 이미지가 크게 보이고, **「다음 이미지」**로 순환하며 전환 시 **우측 방향 슬라이드(또는 동등한 우측 진행 느낌)** 애니메이션이 적용된다.
3. **갤러리 섹션**에서 `images` 목록을 카드 형태로 전부 볼 수 있고, 카드 선택 시 캐러셀의 현재 이미지와 동기화된다(권장 동작).
4. **토글**로 갤러리(또는 지정된 하위 영역)를 보이기/감추기 할 수 있다. 기본값은 “보임”으로 하되, PRD 구현 시 UX 일관성만 유지하면 된다.

## 7. 화면·UI 요구사항 (단일 페이지)

### 7.1 와이어프레임 (논리 레이아웃)

```
┌──────────────────────────────────────────┐
│  {app_title} — 고정 헤더                  │
├──────────────────────────────────────────┤
│                                          │
│     ┌────────────────────────────┐       │
│     │     메인 이미지 (캐러셀)     │       │  ← 우측 슬라이드 전환
│     │   (config의 현재 항목)       │       │
│     └────────────────────────────┘       │
│                                          │
│     [ ▶ 다음 이미지 ]                     │  ← 순환: 마지막 → 첫 이미지
│                                          │
│     [ 눈 아이콘 또는 라벨: 갤러리 표시 ]   │  ← ToggleButton (갤러리 영역)
│     ┌────┐ ┌────┐ ┌────┐                  │
│     │ 1  │ │ 2  │ │ 3  │  …              │  ← ImageGallery (카드)
│     └────┘ └────┘ └────┘                  │
│     (각 카드: 썸네일 + label)              │
└──────────────────────────────────────────┘
```

### 7.2 컴포넌트별 책임

| 컴포넌트 | 책임 |
|----------|------|
| `_layout.tsx` | 전역 레이아웃, 헤더에 `app_title` 표시, 자식 라우트 래핑 |
| `index.tsx` | `config.json` 로드(또는 import), 상태 조립, 하위 컴포넌트 배치 |
| `ImageCarousel.tsx` | 현재 인덱스 이미지 표시, 다음/이전(선택) 전환, **우측 진행 슬라이드** 애니메이션 |
| `ImageGallery.tsx` | `images` 전체를 카드 그리드/리스트로 표시, 항목별 `label` 표시, 선택 시 상위 상태에 알림 |
| `ToggleButton.tsx` | 접근 가능한 토글(스위치/버튼), 갤러리 컨테이너 `display` 또는 높이/opacity 제어와 연동 |

### 7.3 인터랙션·상태

- **캐러셀 인덱스**: 0 … `images.length - 1` 순환.
- **갤러리 ↔ 캐러셀**: 갤러리에서 카드 선택 시 캐러셀 인덱스가 해당 `id` 또는 배열 인덱스와 일치해야 한다.
- **토글**: 갤러리 영역이 숨겨졌을 때 레이아웃이 깨지지 않도록 여백·접근성(스크린 리더 라벨)을 고려한다.

### 7.4 반응형·플랫폼

- 웹: 일반 브라우저 너비에서 가독성 유지(최대 폭 제한·중앙 정렬 권장).
- Android: 터치 타깃 최소 크기 준수, 스크롤 시 헤더 고정 정책은 `_layout`에서 일관되게.

## 12. 수용 기준 (Acceptance Criteria)

1. 웹과 Android에서 **동일한 단일 페이지**가 열리고 헤더 타이틀이 `config.json`의 `app_title`과 같다.
2. 「다음 이미지」만으로 모든 `images` 항목을 순서대로 순환할 수 있다.
3. 전환 애니메이션이 **우측 방향 진행**으로 인지 가능하다.
4. 갤러리에 모든 항목이 카드로 보이며, 각 카드에 `label`이 보인다.
5. 카드를 누르면 캐러셀 이미지가 해당 항목으로 바뀐다.
6. 토글로 갤러리 영역이 보였다 숨겼다 할 수 있다.
7. `images`에서 항목을 제거·추가한 뒤(파일 포함) 앱 코드 변경 없이 목록이 갱신된다.
