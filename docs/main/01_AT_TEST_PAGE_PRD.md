## 2. 제품 한눈에 보기

### 2.1 한 줄 정의

**AT_TEST_PAGE**는 설정 파일 기반으로 이미지 콘텐츠를 손쉽게 바꾸고, 웹에서 쿠폰 데이터를 조회·내려받을 수 있는 테스트/데모용 단일 페이지 애플리케이션이다.

### 2.2 핵심 가치

- **단일 코드베이스**로 웹과 Android를 함께 운영한다.
- 이미지/라벨은 `frontend/env/config.*.json` 수정만으로 변경 가능하다.
- 웹 화면에서 쿠폰 데이터를 조회하고 CSV 다운로드까지 바로 수행할 수 있다.

### 2.3 대상 사용자

- 운영/기획 담당: 이미지 구성·문구 확인, 쿠폰 데이터 샘플 점검
- 개발/QA 담당: 웹·Android 동작 검증, API 연동 검증

## 3. 시스템 구성 요약

### 3.1 아키텍처

```text
[사용자 브라우저 / Android 앱]
            |
            v
[Frontend: Expo Router 단일 화면]
            |
            v
[Backend: FastAPI /api/coupons]
            |
            v
[PostgreSQL: test_coupons_data]
```

### 3.2 환경별 설정

앱/API 동작 값은 역할별 설정 파일에서 분리 관리한다.

- 프론트(웹/앱 화면): `frontend/env/config.dev.json`, `frontend/env/config.prd.json`
- 백엔드(API/DB/CORS): `backend/env/config.dev.json`, `backend/env/config.prd.json`

주요 설정 항목:

- `app_title`: 앱 상단 제목
- `images`: 캐러셀/갤러리 이미지 목록
- `api_url`: 쿠폰 조회 API 주소
- `db.*`: 백엔드 DB 연결 정보
- `cors_origins`: 백엔드 CORS 허용 출처 목록

## 4. 주요 기능

### 4.1 이미지 캐러셀/갤러리

- 메인 이미지 캐러셀에서 **다음 이미지 순환 전환**을 지원한다.
- 갤러리 항목 선택 시 캐러셀 현재 이미지와 동기화된다.
- 갤러리 표시/숨김 토글을 제공한다.

### 4.2 쿠폰 데이터 조회(웹 전용)

- 쿠폰 목록 4개 컬럼을 표 형태로 조회한다.
  - `created`
  - `campaign_label`
  - `workflow_label`
  - `coupon_id`
- 페이지 이동은 다음 방식으로 지원한다.
  - 이전/다음: 커서 기반(keyset)
  - 특정 페이지 입력: 페이지 번호 직접 입력
  - 맨앞/맨뒤 이동
- 현재 조회 범위를 CSV로 다운로드할 수 있다.

### 4.3 공통 UX

- 로딩 상태 표시
- 조회 실패 시 에러 메시지 + 재시도 버튼
- 기본 접근성 라벨(버튼/토글)

## 5. 범위 정의

### 5.1 In Scope (v1.1)

- 웹 + Android 단일 코드베이스 운영
- 이미지 캐러셀/갤러리/토글

### 5.2 Out of Scope (v1.1)

- 데이터 입력/수정/삭제(CUD)
- iOS 전용 배포 파이프라인 상세
- 앱 내 업로드형 CMS 기능

## 6. 수용 기준 (Acceptance Criteria)

1. 앱 상단 타이틀이 `config`의 `app_title` 값과 일치한다.
2. 이미지 목록(`images`)만 수정해도 캐러셀/갤러리 표시 항목이 변경된다.
3. 갤러리에서 선택한 항목이 캐러셀 현재 이미지에 반영된다.
4. 웹에서 쿠폰 목록 조회가 정상 동작한다.
5. 이전/다음·맨앞/맨뒤·직접 입력 페이지 이동이 동작한다.
6. CSV 버튼으로 현재 조회 범위를 다운로드할 수 있다.
7. API 오류 시 에러 안내 및 재시도 동작이 제공된다.
